# 結合試験項目書

| 項目 | 内容 |
|------|------|
| プロジェクト | ハタケル（畑管理アプリ） |
| 対象バージョン | main ブランチ最新 |
| 作成日 | 2026-06-17 |
| 試験環境 | Chrome 最新版 / localhost:5173（`npm run dev`） |

## 試験の目的

各機能モジュール（畑・作物・区画・ログ・イベント・データ永続化）が連携して正しく動作することを確認する。

---

## IT-01 畑管理

### IT-01-01 畑の新規作成

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | ヘッダーの畑名ドロップダウンを開く | 現在の畑名と「＋ 新しい畑」ボタンが表示される |
| 2 | 「＋ 新しい畑」をクリックし、名前を入力して保存 | 畑一覧に追加され、新しい畑がアクティブになる |
| 3 | 畑マップタブを確認 | 新規畑の空グリッドが表示される |
| 4 | ページをリロード | 作成した畑が保持されている（localStorage 永続化確認） |

### IT-01-02 畑の切り替え

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | 畑Aに作物・ログを登録する | - |
| 2 | 畑Bを作成してアクティブにする | 畑マップ・作物・ログが畑B のデータ（空）に切り替わる |
| 3 | 再び畑Aに切り替える | 畑Aのデータが正しく表示される |

### IT-01-03 畑の削除

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | 畑Aに作物・ログ・イベントを登録する | - |
| 2 | 畑Aを削除する | 確認ダイアログが表示される |
| 3 | 削除を確定する | 畑Aが一覧から消え、別の畑がアクティブになる |
| 4 | 作物・ログ・カレンダータブを確認 | 畑Aに紐づいていたデータがすべて消えている |

## IT-06 データ永続化

### IT-06-01 localStorage への自動保存

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | 作物・ログ・イベントを各1件登録する | - |
| 2 | ブラウザのタブを閉じて再度開く | すべてのデータが復元されている |
| 3 | DevTools → Application → Local Storage → `hatake_v2` を確認 | JSON データが保存されている |

### IT-06-02 エクスポートとインポート

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | データを一通り登録した状態でエクスポートを実行 | `hatake-backup-YYYY-MM-DD.json` がダウンロードされる |
| 2 | DevTools でlocalStorage をクリアしてリロード | 空の初期状態になる |
| 3 | エクスポートした JSON ファイルをインポートする | すべてのデータが復元される |
| 4 | 各タブを確認 | 畑・作物・ログ・イベントがエクスポート前の状態と一致する |

## IT-07 タブ間連携

### IT-07-01 作物情報のタブ横断表示

| # | 操作手順 | 期待結果 |
|---|---------|---------|
| 1 | 作物Aを登録（カラー：赤） | - |
| 2 | 畑マップで区画に作物Aを割り当てる | 区画が赤で表示される |

## 合否判定基準

- 全手順で「期待結果」と一致した場合：**合格**
- 1項目でも不一致があった場合：**不合格**（内容を記録して報告）
