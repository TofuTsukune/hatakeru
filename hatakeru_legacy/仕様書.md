# 畑管理アプリ 仕様書

## 概要

ブラウザで動作するシングルページの畑管理アプリ。サーバー不要で、データはすべて `localStorage` に保存する。

- ファイル: `hatake.html`
- 動作環境: モダンブラウザ（Chrome / Edge / Firefox）
- データ保存: `localStorage` キー `hatake`

---

## 画面構成

ヘッダーのタブで4画面を切り替える。

| タブ | ID | 概要 |
|------|----|------|
| 📍 畑マップ | `map` | グリッドで畑レイアウトを管理 |
| 🥬 作物 | `crops` | 作物マスタの一覧・編集 |
| 📋 作業ログ | `logs` | 作業記録の一覧・編集 |
| 📅 カレンダー | `calendar` | 月表示のカレンダー |

---

## データ構造

`localStorage` に JSON 形式で保存。

```json
{
  "crops": [ Crop ],
  "logs":  [ Log ],
  "events": [ Event ],
  "plots": { "行-列": PlotData },
  "gridCols": 8,
  "gridRows": 6
}
```

### Crop（作物）

| フィールド | 型 | 説明 |
|----------|-----|------|
| `id` | string | ユニークID |
| `name` | string | 作物名（必須） |
| `variety` | string | 品種 |
| `status` | `"planned"` / `"growing"` / `"harvested"` | 栽培ステータス |
| `color` | string | 表示カラー（HEX） |
| `notes` | string | メモ |

### PlotData（区画データ）

区画ごとに植付け日・収穫予定日を個別管理する。

| フィールド | 型 | 説明 |
|----------|-----|------|
| `cropId` | string | 割り当て作物のID |
| `plantedDate` | string | 植付け日（YYYY-MM-DD） |
| `expectedHarvest` | string | 収穫予定日（YYYY-MM-DD） |

キーは `"行インデックス-列インデックス"` 形式（例: `"0-1"` = A2）。

### Log（作業ログ）

| フィールド | 型 | 説明 |
|----------|-----|------|
| `id` | string | ユニークID |
| `date` | string | 作業日（YYYY-MM-DD） |
| `type` | string | 作業種別（下記参照） |
| `cropId` | string / null | 関連作物ID |
| `notes` | string | メモ |

作業種別: `water`（水やり）/ `fertilize`（施肥）/ `harvest`（収穫）/ `weed`（草取り）/ `plant`（植付け）/ `other`（その他）

### Event（予定）

| フィールド | 型 | 説明 |
|----------|-----|------|
| `id` | string | ユニークID |
| `date` | string | 予定日（YYYY-MM-DD） |
| `title` | string | タイトル（必須） |
| `type` | string | 種別（Log の作業種別と共通） |
| `cropId` | string / null | 関連作物ID |
| `done` | boolean | 完了フラグ |

## 機能仕様

### 畑マップ

#### グリッド表示

- デフォルト: 8列 × 6行（最大 16列 × 12行）
- セルサイズ: 幅68px × 高92px
- セルラベル: 行をアルファベット（A〜）、列を数字（1〜）で表示（例: A1, B3）
- 作物が割り当てられたセル: 作物カラーで背景・枠を着色
- 日付が設定されている場合: セル内に `植:M/D`・`収:M/D` を表示

#### セル操作

セルをクリックするとモーダルが開く。

| 状態 | モーダルの内容 |
|------|--------------|
| 空セル | 作物選択のみ |
| 作物あり | 作物選択 ＋ 作物情報編集フォーム |

作物情報編集フォームの項目:
- 作物名（必須）
- 品種
- 植付け日（この区画専用）
- 収穫予定日（この区画専用）
- ステータス
- メモ

> 植付け日・収穫予定日は **区画ごと** に独立して設定する。同じ作物（例: トマト）を複数区画に植えた場合、それぞれ異なる日付を持てる。

作物選択を変更した場合、日付フィールドはクリアされる（前の作物の日付は引き継がない）。

#### グリッドサイズ変更

「グリッドサイズ変更」ボタンから列数・行数を変更できる。縮小した場合、範囲外の区画データは削除される。

### 作物

作物マスタの管理画面。

#### 一覧表示

登録済み作物をカード形式で表示。カード内容:
- 作物名・品種
- ステータスバッジ（予定 / 栽培中 / 収穫済み）
- メモ
- 編集ボタン・作業記録ボタン

#### 作物の追加・編集

「＋ 作物を追加」または編集ボタンからモーダルを開く。

入力項目:
- 作物名（必須）
- 品種
- ステータス
- カラー（10色から選択）
- メモ

> 植付け日・収穫予定日は畑マップの区画ごとに設定するため、作物マスタには持たない。

#### 作物の削除

作物を削除すると、関連する作業ログ・予定・区画割り当てもすべて削除される。

---

### 作業ログ

#### 一覧表示

作業日の降順で表示。フィルター:
- 作物で絞り込み
- 作業種別で絞り込み

#### 作業の追加・編集

入力項目:
- 日付（必須）
- 作業種別（必須）
- 作物（任意）
- メモ

---

### カレンダー

月単位の表示。◀ ▶ ボタンで月を移動する。

#### 表示内容

各日付セルに以下を重ねて表示する:

| 種別 | 色 | 内容 |
|------|----|------|
| 区画の植付け日 | 緑 | `🌱 植付け: 作物名 (区画)` |
| 区画の収穫予定日 | オレンジ | `🌾 収穫予定: 作物名 (区画)` |
| 作業ログ | 緑（薄） | `アイコン 作業種別` |
| 予定 | 青 | タイトル（完了済みは取り消し線） |

> 同じ作物を複数区画に植えた場合、区画番号（例: A2, A3）で区別して表示する。

#### 予定の追加・編集

カレンダーの日付セルをクリック、または「＋ 予定を追加」ボタンから追加できる。

入力項目:
- 日付（必須）
- タイトル（必須）
- 種別
- 関連作物（任意）

## UI共通仕様

- テーマ: ダークテーマ固定
- モーダル: オーバーレイ背景クリックで閉じる
- ボタン種別: Primary（青）/ Danger（赤）/ Ghost（グレー）
- 空状態: 一覧が空の場合はアイコンとメッセージを表示
